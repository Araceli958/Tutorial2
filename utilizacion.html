"USO DE LOS COMANDOS GIT"

git init

El comando git init crea un nuevo repositorio de Git. Al ejecutar git init, se crea un subdirectorio de .git en el directorio de trabajo actual, que contiene todos los metadatos de Git necesarios para el nuevo repositorio.

git status

Enumera los archivos que se han preparado, los que están sin preparar y los archivos sin seguimiento. Los mensajes de estado también incluyen información importante para preparar archivos o deshacer su preparación.

git add

Su uso principal es recibir tanto una ruta de archivo como de un directorio; si es de un directorio, el comando añade recursivamente los archivos que están dentro de él.

git config

Al ejecutar git config, se modificará un archivo de texto de configuración. El caso práctico más básico de este comando es invocarlo con un nombre de configuración.

git commit

Su funcion es guardar todos los cambios hechos en la área de preparación, junto con una breve descripción del usuario

git log

Se usa para ver las versiones creadas, se mostrara informacion como:

Identificador del commit

Autor

Fecha de realización

Mensaje enviado

git clone

Se utiliza principalmente para apuntar a un repositorio existente y clonar o copiar dicho repositorio en un nuevo directorio, en otra ubicación. La clonación crea automáticamente una conexión remota llamada "origin" que apunta al repositorio original.

git diff

Sirve para ver las diferencias con los archivos anteriores de los archivos que no se han añadido al área de ensayo. Esto código muestra exactamente los cambios que se han realizado dentro de los archivos. Se suele utilizar después de ver los cambios que ha habido en los archivos.

git checkout

Usaremos el comando git checkout principalmente para cambiarte de una rama a otra. También lo podemos usar para chequear archivos y commits.

git restore

Es una herramienta para revertir cambios no comprometidos. Se puede usar en tres modos diferentes, dependiendo de si desea revertir el trabajo en la copia de trabajo, en el índice o en ambos.

git branch

Para saber qué ramas están disponibles y cuál es el nombre de la rama actual, ejecuta git branch. hay otras

Nuevas ramas

git checkout -b ＜new-branch＞: se crea y se extrae la rama simultáneamente.

git fetch

Puedes usar git fetch para saber los cambios hechos en el repo/rama remoto desde el último pull. Esto es útil para permitir la comprobación antes de hacer un pull real.

git merge

La herramienta git merge se utiliza para fusionar uno o más ramas dentro de la rama que tienes activa.

git push

Se suele usar para publicar y cargar cambios locales a un repositorio central. Después de modificar el repositorio local, se ejecuta un comando push para compartir las modificaciones con miembros remotos del equipo.

git pull

El comando git pull descargará todos los cambios desde el punto de separación de la rama local y la rama maestra.

Ejemplo

git pull remote: Recupera la copia del origen remoto especificado de la rama actual y fusiónala de inmediato en la copia local.

git pull --no-commit remote: extrae el contenido remoto, pero no crea una nueva confirmación de fusión.

git rm

Su uso es eliminar los archivos del area de ensayo, no solo los archivos que son rastreados por git, sino que también los elimina del área de trabajo.

git switch

Este comando su funcion es Cambiar a una rama específica. El árbol de trabajo y el índice se actualizan para que coincidan con la rama.

git remote

Para ver los remotos que se tienen configurados, se debe ejecutar el comando git remote mostrando los nombres de cada uno de los remotos que tienen especificados.
